# EGCT 개인정보 보호 정책

## 원칙: 개인정보 최소화

> "P2P 거래에는 공개키만 필요합니다. 개인정보는 불필요합니다."

## 개인정보 라이프사이클

### 1단계: 초기 할당 (임시 보관)
```
목적: 등록 안내 이메일 발송
보관 정보:
  • 이름
  • 이메일
  • 할당량
  • 임시 공개키
```

### 2단계: 등록 프로세스
```
보유자 행동:
  1. 등록 링크 클릭
  2. 자신의 지갑 생성 (BIP39)
  3. 실제 공개키 등록
```

### 3단계: 개인정보 파기 (등록 완료 후)
```
파기 정보:
  ❌ 이름
  ❌ 이메일
  ❌ 가입일
  ❌ 등록 토큰
  ❌ 등록 링크

보존 정보:
  ✅ 공개키 (익명)
  ✅ 잔액
  ✅ UTXO
```

## 파기 시점

### 자동 파기 조건
1. ✅ 모든 보유자 등록 완료
2. ✅ 30일 등록 기간 만료

### 수동 파기
관리자가 언제든지 실행 가능:
```bash
cd /var/www/government-ai-systems/currency/data
python3 privacy_cleanup.py
```

## 파기 후 시스템 작동

### ✅ 가능한 것
- P2P 거래 (공개키만 사용)
- 잔액 조회
- UTXO 관리
- 총 발행량 검증
- 거래 내역 조회

### ❌ 불가능한 것
- 보유자 신원 확인
- 이메일 발송
- 개인정보 조회

## 장점

### 1. 개인정보보호법 완벽 준수
```
제21조(개인정보의 파기)
① 개인정보처리자는 보유기간의 경과, 
   개인정보의 처리 목적 달성 등 
   그 개인정보가 불필요하게 되었을 때에는 
   지체 없이 그 개인정보를 파기하여야 한다.
```

### 2. GDPR 준수
- Data Minimization (최소화)
- Purpose Limitation (목적 제한)
- Storage Limitation (보관 제한)

### 3. 보안 강화
- 해킹 시에도 개인정보 없음
- 데이터 유출 위험 제로

### 4. 진정한 탈중앙화
- 비트코인처럼 완전한 익명성
- 공개키만으로 거래
- 중앙 데이터베이스 불필요

## 백업 정책

개인정보 파기 전 자동 백업:
```
backup_before_cleanup_20251208_123456/
├── holders.json (원본)
├── registration_tokens.json
├── registration_links.csv
└── email_template.*
```

**보관 기간**: 30일
**보관 장소**: 서버 로컬 (암호화 권장)
**접근 권한**: 관리자만

## 파기 로그

모든 파기 작업은 로그에 기록:
```json
{
  "cleanupDate": "2025-12-08T12:34:56",
  "totalHolders": 70,
  "registeredAtCleanup": 70,
  "filesDestroyed": [...],
  "backupLocation": "backup_before_cleanup_..."
}
```

## 비교: 기존 vs 개선

| 구분 | 기존 방식 | 개선 방식 |
|------|----------|----------|
| 개인정보 | 영구 보관 | 등록 후 파기 |
| 보안 위험 | 높음 | 낮음 |
| 법적 준수 | 불완전 | 완벽 |
| 거래 방식 | 동일 | 동일 |
| 프라이버시 | 낮음 | 높음 |

## 결론

✅ **P2P 거래 = 공개키만 필요**
✅ **개인정보 = 초기 등록 후 즉시 파기**
✅ **보안 = 최대화**
✅ **법적 준수 = 완벽**

이것이 진정한 **개인정보 최소화 원칙**입니다.
